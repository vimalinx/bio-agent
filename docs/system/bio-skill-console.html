这是一个静态但可读的控制台原型，用真实 CLI 输出把 request -> plans -> approved plan -> draft -> run-status -> run-review 这条主链直接摊开。它的目标不是替代真正的前端，而是把当前控制平面讲清楚。

命令链路

1. 计划阶段 用户需求先归一化，再给两个方案做 trade-off。 propose review approve edit
2. 执行准备 已批准 plan 冻结下来，再展开成候选 skill 草稿。 draft run-init
3. 运行阶段 单 stage 推进，必要时暂停，再给人或 agent 一个可读摘要。 run-next-stage run-pause run-resume run-status run-review
